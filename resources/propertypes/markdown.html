代码重构方案

正如上面提到的，对于配置文件的读取，不要直接使用 process.env 进行到处散落的调用，这极易引发错误。我为你生成了一个单例模式的设计：

TypeScript 
import { z } from 'zod';

// 1. Define schema validation for environment variables
const envSchema = z.object({
  DATABASE_URL: z.string().url(),
  JWT_SECRET: z.string().min(32),
  PORT: z.coerce.number().default(3000),
});

// 2. Export validated config
export const config = envSchema.parse(process.env);
