`bash` 工具的实现主要分布在 `rust/crates/runtime/src/bash.rs`（核心逻辑）和相关的 runtime 模块中。

## 核心实现文件

### 1. `rust/crates/runtime/src/bash.rs`

这是 `bash` 工具的核心实现文件，包含以下关键组件：

#### 数据结构
- `BashCommandInput` - 输入参数结构体，包含命令、超时、描述、后台运行标志、沙箱配置等 [1](#8-0) 
- `BashCommandOutput` - 输出结果结构体，包含 stdout、stderr、中断标志、后台任务 ID、沙箱状态等 [2](#8-1) 

#### 核心函数
- `execute_bash()` - 同步执行入口，处理后台任务和异步执行的分发 [3](#8-2) 
- `execute_bash_async()` - 异步执行实现，处理超时、命令执行和结果处理 [4](#8-3) 

### 2. `rust/crates/runtime/src/bash_validation.rs`

Bash 验证子模块，包含 9 个验证模块（Lane 1 功能）：
- `validate_read_only()` - 阻止只读模式下的写操作
- `validate_mode()` - 验证权限模式
- `validate_paths()` - 路径验证
- `validate_sed()` - sed 命令验证
- `destructive_command_warning()` - 破坏性命令警告
- `command_semantics()` - 命令语义分类

### 3. `rust/crates/runtime/src/lib.rs`

导出 bash 相关的公共 API [5](#8-4) 。

## 实现流程

```mermaid
graph TD
    A[Tool Call: bash] --> B[execute_bash in bash.rs]
    B --> C{run_in_background?}
    C -->|Yes| D[Spawn background process]
    C -->|No| E[execute_bash_async]
    E --> F[sandbox_status_for_input - 计算沙箱状态]
    F --> G[detect_and_emit_ship_prepared - Git 推送检测]
    G --> H[prepare_tokio_command - 准备命令]
    H --> I{timeout set?}
    I -->|Yes| J[Execute with timeout]
    I -->|No| K[Execute normally]
    J --> L{timeout exceeded?}
    L -->|Yes| M[Return timeout output]
    L -->|No| N[Process output]
    K --> N
    N --> O[truncate_output - 截断输出]
    O --> P[Return BashCommandOutput]
    D --> Q[Return background task ID]
```

## 核心实现逻辑

### 后台任务处理
当 `run_in_background` 为 true 时，bash 工具会创建一个后台进程并返回任务 ID：

```rust
if input.run_in_background.unwrap_or(false) {
    let mut child = prepare_command(&input.command, &cwd, &sandbox_status, false);
    let child = child
        .stdin(Stdio::null())
        .stdout(Stdio::null())
        .stderr(Stdio::null())
        .spawn()?;

    return Ok(BashCommandOutput {
        background_task_id: Some(child.id().to_string()),
        no_output_expected: Some(true),
        // ...
    });
}
``` [6](#8-5) 

### 超时处理
支持命令超时，超时后返回结构化的超时输出 [7](#8-6) 。

### 沙箱集成
通过 `sandbox_status_for_input()` 计算沙箱状态，支持 Linux 命名空间隔离和 macOS 的有限支持 [8](#8-7) 。

### Git 推送检测
`detect_and_emit_ship_prepared()` 检测 git push 操作并发出相应的 provenance 事件 [9](#8-8) 。

```rust
pub fn execute_bash(input: BashCommandInput) -> io::Result<BashCommandOutput> {
    let cwd = env::current_dir()?;
    let sandbox_status = sandbox_status_for_input(&input, &cwd);

    if input.run_in_background.unwrap_or(false) {
        let mut child = prepare_command(&input.command, &cwd, &sandbox_status, false);
        let child = child
            .stdin(Stdio::null())
            .stdout(Stdio::null())
            .stderr(Stdio::null())
            .spawn()?;

        return Ok(BashCommandOutput {
            stdout: String::new(),
            stderr: String::new(),
            raw_output_path: None,
            interrupted: false,
            is_image: None,
            background_task_id: Some(child.id().to_string()),
            backgrounded_by_user: Some(false),
            assistant_auto_backgrounded: Some(false),
            dangerously_disable_sandbox: input.dangerously_disable_sandbox,
            return_code_interpretation: None,
            no_output_expected: Some(true),
            structured_content: None,
            persisted_output_path: None,
            persisted_output_size: None,
            sandbox_status: Some(sandbox_status),
        });
    }

    let runtime = Builder::new_current_thread().enable_all().build()?;
    runtime.block_on(execute_bash_async(input, sandbox_status, cwd))
}

```rust
async fn execute_bash_async(
    input: BashCommandInput,
    sandbox_status: SandboxStatus,
    cwd: std::path::PathBuf,
) -> io::Result<BashCommandOutput> {
    // Detect and emit ship provenance for git push operations
    detect_and_emit_ship_prepared(&input.command);

    let mut command = prepare_tokio_command(&input.command, &cwd, &sandbox_status, true);

    let output_result = if let Some(timeout_ms) = input.timeout {
        if let Ok(result) = timeout(Duration::from_millis(timeout_ms), command.output()).await {
            (result?, false)
        } else {
            return Ok(timeout_output(&input, timeout_ms, sandbox_status));
        }
    } else {
        (command.output().await?, false)
    };

    let (output, interrupted) = output_result;
    let stdout = truncate_output(&String::from_utf8_lossy(&output.stdout));
    let stderr = truncate_output(&String::from_utf8_lossy(&output.stderr));
    let no_output_expected = Some(stdout.trim().is_empty() && stderr.trim().is_empty());
    let return_code_interpretation = output.status.code().and_then(|code| {
        if code == 0 {
            None
        } else {
            Some(format!("exit_code:{code}"))
        }
    });

    Ok(BashCommandOutput {
        stdout,
        stderr,
        raw_output_path: None,
        interrupted,
        is_image: None,
        background_task_id: None,
        backgrounded_by_user: None,
        assistant_auto_backgrounded: None,
        dangerously_disable_sandbox: input.dangerously_disable_sandbox,
        return_code_interpretation,
        no_output_expected,
        structured_content: None,
        persisted_output_path: None,
        persisted_output_size: None,
        sandbox_status: Some(sandbox_status),
    })
}
```
